#### <a name="issue"></a><span data-ttu-id="5c3f3-119">问题</span><span class="sxs-lookup"><span data-stu-id="5c3f3-119">Issue</span></span>
<span data-ttu-id="5c3f3-120">如果计算机的 Internet 访问受限（如 CI/CD 方案中的生成计算机），安装/还原签名的 nuget 包将导致警告 ([NU3028](https://docs.microsoft.com/en-us/nuget/reference/errors-and-warnings/nu3028))，因为吊销服务器不可访问。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-120">If the machine has restricted internet access (such as a build machine in a CI/CD scenario), installing/restoring a signed nuget package will result a warning ([NU3028](https://docs.microsoft.com/en-us/nuget/reference/errors-and-warnings/nu3028)) since the revocation servers are not reachable.</span></span> <span data-ttu-id="5c3f3-121">这是预期情况。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-121">This is expected.</span></span> <span data-ttu-id="5c3f3-122">但是，在某些情况下，这可能会产生意外的结果，如安装/还原包花费的时间比平常长。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-122">However, in some cases, this may have unintended concequences such as the package install/restore taking longer than usual.</span></span>

#### <a name="workaround"></a><span data-ttu-id="5c3f3-123">解决方法</span><span class="sxs-lookup"><span data-stu-id="5c3f3-123">Workaround</span></span>
<span data-ttu-id="5c3f3-124">更新到 Visual Studio 15.8.4 和 NuGet.exe 4.8.1，我们在其中引入了一个环境变量来切换吊销检查模式。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-124">Update to Visual Studio 15.8.4 and NuGet.exe 4.8.1 where we introduced an environment variable to switch the revocation check mode.</span></span>
<span data-ttu-id="5c3f3-125">将 `NUGET_CERT_REVOCATION_MODE` 环境变量设置为 `offline` 将迫使 NuGet 仅针对缓存证书吊销列表检查证书吊销状态，NuGet 不会尝试访问吊销服务器。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-125">Setting the `NUGET_CERT_REVOCATION_MODE` environment variable to `offline` will force NuGet to check the revocation status of the certificate only against the cached certificate revocation list, and NuGet will not attempt to reach revocation servers.</span></span> <span data-ttu-id="5c3f3-126">如果吊销检查模式设置为 `offline`，警告将降级为信息。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-126">When the revocation check mode is set to `offline`, the warning will be downgraded to an info.</span></span>

> [!Warning]
> <span data-ttu-id="5c3f3-127">不建议在正常情况下将吊销检查模式切换到脱机。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-127">It is not recommended to switch the revocation check mode to offline under normal cirumstances.</span></span> <span data-ttu-id="5c3f3-128">执行此操作将导致 NuGet 跳过联机吊销检查，并只对可能过期的缓存证书吊销列表执行脱机吊销检查。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-128">Doing so will cause NuGet to skip online revocation check and perform only an offline revocation check against the cached certificate revocation list which may be out of date.</span></span> <span data-ttu-id="5c3f3-129">这意味着将继续安装/还原签名证书可能已经吊销的包，否则吊销检查将失败，并且不会安装。</span><span class="sxs-lookup"><span data-stu-id="5c3f3-129">This means packages where the signing certificate may have been revoked, will continue to be installed/restored, which otherwise would have failed revocation check and would not have been installed.</span></span>
