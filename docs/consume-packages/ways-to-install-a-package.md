## <a name="what-happens-when-a-package-is-installed"></a><span data-ttu-id="b3763-128">安装包时会发生什么情况</span><span class="sxs-lookup"><span data-stu-id="b3763-128">What happens when a package is installed</span></span>

<span data-ttu-id="b3763-129">简而言之，不同的 NuGet 工具通常会对项目文件或 `packages.config` 中的包创建引用，然后执行包还原，从而有效安装包。</span><span class="sxs-lookup"><span data-stu-id="b3763-129">Simply said, the different NuGet tools typically create a reference to a package in the project file or `packages.config`, then perform a package restore, which effectively installs the package.</span></span> <span data-ttu-id="b3763-130">`nuget install` 除外，它仅将包展开到 `packages` 文件夹，而不修改任何其他文件。</span><span class="sxs-lookup"><span data-stu-id="b3763-130">The exception is `nuget install`, which only expands the package into a `packages` folder and does not modify any other files.</span></span>

<span data-ttu-id="b3763-131">一般流程如下：</span><span class="sxs-lookup"><span data-stu-id="b3763-131">The general process is as follows:</span></span>

1. <span data-ttu-id="b3763-132">（除 `nuget.exe` 之外的所有工具）将包标识符和版本记录到项目文件或 `packages.config`中。</span><span class="sxs-lookup"><span data-stu-id="b3763-132">(All tools except `nuget.exe`) Record the package identifer and version into the project file or `packages.config`.</span></span>

2. <span data-ttu-id="b3763-133">获取包：</span><span class="sxs-lookup"><span data-stu-id="b3763-133">Acquire the package:</span></span>
   - <span data-ttu-id="b3763-134">检查包是否（通过标识符和版本号）已安装在 global-packages 文件夹中，如[管理全局包和缓存文件夹](managing-the-global-packages-and-cache-folders.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="b3763-134">Check if the package (by exact identifer and version number) is already installed in the *global-packages* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

   - <span data-ttu-id="b3763-135">如果包不在 global-packages 文件夹中，请尝试从[配置文件](Configuring-NuGet-Behavior.md)中列出的源检索包。</span><span class="sxs-lookup"><span data-stu-id="b3763-135">If the package is not in the *global-packages* folder, attempt to retrieve it from the sources listed listed in the [configuration files](Configuring-NuGet-Behavior.md).</span></span> <span data-ttu-id="b3763-136">对于在线源，请首先尝试从缓存中检索包，除非通过 `nuget.exe` 命令指定 `-NoCache` 或通过 `dotnet restore` 指定 `--no-cache`。</span><span class="sxs-lookup"><span data-stu-id="b3763-136">For online sources, attempt first to retrieve the package from the cache unless `-NoCache` is specified with `nuget.exe` commands or `--no-cache` is specified with `dotnet restore`.</span></span> <span data-ttu-id="b3763-137">（Visual Studio 和 `dotnet add package` 始终使用缓存。）如果从缓存中使用包，“缓存”将出现在输出中。</span><span class="sxs-lookup"><span data-stu-id="b3763-137">(Visual Studio and `dotnet add package` always use the cache.) If a package is used from the cache, "CACHE" appears in the output.</span></span> <span data-ttu-id="b3763-138">缓存有 30 分钟的到期时间。</span><span class="sxs-lookup"><span data-stu-id="b3763-138">The cache has an expiration time of 30 minutes.</span></span>

   - <span data-ttu-id="b3763-139">如果包不在缓存中，请尝试从配置中列出的源下载包。</span><span class="sxs-lookup"><span data-stu-id="b3763-139">If the package is not in the cache, attempt to download it from the sources listed in the configuration.</span></span> <span data-ttu-id="b3763-140">如果下载包，则会在输出中出现“GET”和“OK”。</span><span class="sxs-lookup"><span data-stu-id="b3763-140">If a package is downloaded, "GET" and "OK" appear in the output.</span></span>

   - <span data-ttu-id="b3763-141">如果无法从任何源成功获取包，安装将失败，并显示诸如 [NU1103](../reference/errors-and-warnings/NU1103.md) 之类的错误。</span><span class="sxs-lookup"><span data-stu-id="b3763-141">If the package cannot be successfully acquired from any sources, installation fails at this point with an error such as [NU1103](../reference/errors-and-warnings/NU1103.md).</span></span> <span data-ttu-id="b3763-142">注意，来自 `nuget.exe` 命令的错误仅显示最后检查的源，但意味着无法从任何源获取包。</span><span class="sxs-lookup"><span data-stu-id="b3763-142">Note that errors from `nuget.exe` commands show only the last source checked, but implies that the package wasn't available from any source.</span></span>

   <span data-ttu-id="b3763-143">获取包时，NuGet 配置中的源顺序可能适用：</span><span class="sxs-lookup"><span data-stu-id="b3763-143">When acquiring the package, the order of sources in the NuGet configuration may apply:</span></span>

   - <span data-ttu-id="b3763-144">对于使用 PackageReference 格式的项目，NuGet 将首先检查源本地文件夹和网络共享，然后再检查 HTTP 源。</span><span class="sxs-lookup"><span data-stu-id="b3763-144">For projects using the PackageReference format, NuGet checks sources local folder and network shares before checking HTTP sources.</span></span>

   - <span data-ttu-id="b3763-145">对于使用 `packages.config` 管理格式的项目，NuGet 会使用配置中的源顺序。</span><span class="sxs-lookup"><span data-stu-id="b3763-145">For projects using the `packages.config` management format, NuGet uses the order of the sources in the configuration.</span></span> <span data-ttu-id="b3763-146">还原操作除外，这种情况下将忽略源排序，NuGet 会使用任何最先响应的源中的包。</span><span class="sxs-lookup"><span data-stu-id="b3763-146">An exception is restore operations, in which case source ordering is ignored and NuGet uses the package from whichever source responds first.</span></span>

   - <span data-ttu-id="b3763-147">一般情况下，NuGet 检查源的顺序不具有特别意义，因为具有特定标识符和版本号的任意给定包在找到的源方面完全相同。</span><span class="sxs-lookup"><span data-stu-id="b3763-147">In general, the order in which NuGet checks sources isn't particularly meaningful, because any given package with a specific identifier and version number is exactly the same on whatever source it's found.</span></span>

3. <span data-ttu-id="b3763-148">（除 `nuget.exe` 之外的所有工具）保存包副本和 http-cache 文件夹中的其他信息，如[管理全局包和缓存文件夹](managing-the-global-packages-and-cache-folders.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="b3763-148">(All tools except `nuget.exe`) Save a copy of the package and other information in the *http-cache* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

4. <span data-ttu-id="b3763-149">如下载，请将包安装到每个用户的 global-packages 文件夹中。</span><span class="sxs-lookup"><span data-stu-id="b3763-149">If downloaded, install the package into the per-user *global-packages* folder.</span></span> <span data-ttu-id="b3763-150">NuGet 创建每个包标识符的子文件夹，然后创建每个已安装包版本的子文件夹。</span><span class="sxs-lookup"><span data-stu-id="b3763-150">NuGet creates a subfolder for each package identifier, then creates subfolders for each installed version of the package.</span></span>

5. <span data-ttu-id="b3763-151">更新其他项目文件和文件夹：</span><span class="sxs-lookup"><span data-stu-id="b3763-151">Update other project files and folders:</span></span>

    - <span data-ttu-id="b3763-152">对于使用 PackageReference 的项目，更新存储在 `obj/project.assets.json` 中的包依赖项关系图。</span><span class="sxs-lookup"><span data-stu-id="b3763-152">For projects using PackageReference, update the package dependency graph stored in `obj/project.assets.json`.</span></span> <span data-ttu-id="b3763-153">包内容本身不会复制到任何项目文件夹中。</span><span class="sxs-lookup"><span data-stu-id="b3763-153">Package contents themselves are not copied into any project folder.</span></span>
    - <span data-ttu-id="b3763-154">对于使用 `packages.config` 的项目，将展开包中与项目目标框架匹配的那些部分复制到项目的 `packages` 文件夹。</span><span class="sxs-lookup"><span data-stu-id="b3763-154">For projects using `packages.config`, copy those parts of the expanded package that match the project's target framework into project's `packages` folder.</span></span> <span data-ttu-id="b3763-155">（在使用 `nuget install` 时，将复制整个展开包，因为 `nuget.exe` 不检查项目文件来标识目标框架。）</span><span class="sxs-lookup"><span data-stu-id="b3763-155">(When using `nuget install`, the entire expanded package is copied because `nuget.exe` does not examine project files to identify the target framework.)</span></span>
    - <span data-ttu-id="b3763-156">如果包使用[源和配置文件转换](../create-packages/source-and-config-file-transformations.md)，则更新 `app.config` 和/或 `web.config`。</span><span class="sxs-lookup"><span data-stu-id="b3763-156">Update `app.config` and/or `web.config` if the package uses [source and config file transformations](../create-packages/source-and-config-file-transformations.md).</span></span>

6. <span data-ttu-id="b3763-157">安装任何低级别的依赖项（如果项目中尚不存在）。</span><span class="sxs-lookup"><span data-stu-id="b3763-157">Install any down-level dependencies if not already present in the project.</span></span> <span data-ttu-id="b3763-158">此过程可能会更新过程中的包版本，如[依赖项解析](../consume-packages/dependency-resolution.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="b3763-158">This process might update package versions in the process, as described in [Dependency Resolution](../consume-packages/dependency-resolution.md).</span></span>

7. <span data-ttu-id="b3763-159">（仅适用于 Visual Studio）如果可用，请在 Visual Studio 窗口中显示包的自述文件。</span><span class="sxs-lookup"><span data-stu-id="b3763-159">(Visual Studio only) Display the package's readme file, if available, in a Visual Studio window.</span></span>
