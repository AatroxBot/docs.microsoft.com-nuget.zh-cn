<span data-ttu-id="a4a1e-420">本部分介绍的客户端概念，尽管不一定是托管协议，但应为任何实际目录客户端实现的一部分。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-420">This section describes a client concept that, although is not necessarily mandated by the protocol, should be part of any practical catalog client implementation.</span></span>

<span data-ttu-id="a4a1e-421">因为目录是时间索引的仅限追加的数据结构，应存储在客户端**游标**本地，最多的时间点在客户端表示已处理目录项。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-421">Because the catalog is an append-only data structure indexed by time, the client should store a **cursor** locally, representing up to what point in time the client has processed catalog items.</span></span> <span data-ttu-id="a4a1e-422">请注意，应永远不会使用客户端的计算机时钟生成此游标值。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-422">Note that this cursor value should never be generated using the client's machine clock.</span></span> <span data-ttu-id="a4a1e-423">相反，值应来自于目录对象的`commitTimestamp`值。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-423">Instead, the value should come from a catalog object's `commitTimestamp` value.</span></span>

<span data-ttu-id="a4a1e-424">每次客户端想要处理的包源上的新事件，它需要仅查询目录的提交时间戳的所有目录项大于其存储的游标。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-424">Every time the client wants to process new events on the package source, it need only query the catalog for all catalog items with a commit timestamp greater than its stored cursor.</span></span> <span data-ttu-id="a4a1e-425">客户端已成功处理所有新的目录项后，它会记录作为它的新游标值只是处理目录项的最新的提交时间戳。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-425">After the client successfully processes all new catalog items, it records the latest commit timestamp of catalog items just processed as its new cursor value.</span></span>

<span data-ttu-id="a4a1e-426">使用此方法，而客户端可以确保永远不会错过包源发生的任何包事件。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-426">Using this approach, the client can be sure to never miss any package events that occurred on the package source.</span></span>
<span data-ttu-id="a4a1e-427">此外，客户端绝不会重新处理在游标的记录的提交时间戳之前的旧事件。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-427">Additionally, the client never has to reprocess old events prior to the cursor's recorded commit timestamp.</span></span>

<span data-ttu-id="a4a1e-428">这一功能强大的游标概念用于许多 nuget.org 后台作业，用于使 V3 API 本身保持最新。</span><span class="sxs-lookup"><span data-stu-id="a4a1e-428">This powerful concept of cursors is used for many of nuget.org background jobs and is used to keep the V3 API itself up-to-date.</span></span>
